# Gift catalog — default gestures (Russian)

~12 default non-material gestures. Russian name + 1-line warm description. Tone check: read each as if saying it to a tired partner; rewrite if it sounds like an HR email.

These are defaults available to every pair at launch. Pair can also add custom free-form gestures (Pro = saved to pair's catalog). All non-material — no shopping links, no scheduling to a future date (it's a promise, redeem whenever).

---

1. **Завтрак в постель**
   — Утром я всё принесу сам(а). Тебе можно валяться.

2. **Массаж**
   — 15 минут, без спешки. Плечи, спина — как скажешь.

3. **Ты выбираешь фильм**
   — Сегодня вечером выбираешь ты. Я не ворчу.

4. **Домашний ужин**
   — Я готовлю. Ты просто садишься за стол.

5. **Тихий час для тебя**
   — Один час без дел и меня. Я беру на себя всё остальное.

6. **Прогулка вдвоём**
   — Просто идём куда глаза глядят. Без плана.

7. **Кофе с утра готов**
   — Просыпаешься — кофе уже ждёт. Как любишь.

8. **Ты спишь, я убираю**
   — Пока ты отдыхаешь, я разгребаю квартиру.

9. **Ванна для тебя**
   — Я набираю. Полотенце, свечи, покой. Время твоё.

10. **Плейлист вечера — твой**
    — Весь вечер звучит то, что хочешь ты. Без моих вето.

11. **Обнимашки по требованию**
    — Без повода, без разговоров. Просто подойди.

12. **Сладкое за тебя**
    — Сегодня за десертом хожу я. Какое твоё?

---

## Tone check (why each passed)
- Each names a concrete act, not an abstraction ("хочу порадовать тебя" = HR email = rewrite).
- Each removes a load from the partner (food, chore, decision, attention), not a performance.
- Each feels sayable to a tired partner without eye-roll.

## Out of scope (by product rule)
- Material gifts / shopping links — explicitly non-material.
- Scheduling to a specific future date — it's a promise, redeem whenever.
- Reminders that nag the giver — anxiety risk.

## Notes
- Catalog is shared per pair; a custom gesture added by one is available to both.
- Same gift sent twice = two ledger entries (allowed).
- Decline copy (see docs/copy/gifts.md) is warm: "пропустил(а)" / "всё в порядке", never "rejected".
